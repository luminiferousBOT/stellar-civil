━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

━━ MODULE 1.1 — TWO-WAY / ONE-WAY SLAB (IS 456:2000, Annex D) ━━

Inputs:
- Lx (shorter span, m), Ly (longer span, m)
- fck (dropdown: M20/M25/M30/M35), fy (dropdown: Fe415/Fe500/Fe550)
- Exposure condition (Mild/Moderate/Severe/Very Severe/Extreme)
- Live load (kN/m²), Floor finish load (kN/m²), Partition load (kN/m²)
- Panel type (9 types as per IS 456 Annex D Table 26)
- Support condition auto-determines if one-way (Ly/Lx > 2) or two-way

Logic:
Step 1: Determine slab type (Ly/Lx > 2 → one-way)
Step 2: Assume D, compute d = D - cover - φ/2
Step 3: Self-weight of slab = 25 × D kN/m²
Step 4: Total factored load wu = 1.5 × (DL + LL)
Step 5: For two-way: Read αx, αy moment coefficients from IS 456 Table 26
based on panel type and Ly/Lx ratio. Interpolate between values.
Step 6: Mx = αx × wu × Lx², My = αy × wu × Lx²
Step 7: Check depth: dreq = √(M / (0.138 × fck × b))
Step 8: Ast_x = M_x / (0.87 × fy × d × (1 - (Ast×fy)/(b×d×fck)))
Solve quadratic for exact Ast
Step 9: Spacing = (area of 1 bar × 1000) / Ast
Step 10: Check min steel = 0.12% bD (Fe415) per Cl. 26.5.2.1
Step 11: Check max spacing ≤ 3d or 300mm per Cl. 26.3.3
Step 12: Check L/d ratio per IS 456 Cl. 23.2.1
Step 13: Shear check at d from support
Step 14: Development length check per Cl. 26.2.1

Output:
- D provided, d effective
- Ast_x required, provided (bar dia + spacing)
- Ast_y required, provided
- Distribution steel (0.12% for HYSD)
- All 14 step table entries

━━ MODULE 1.2 — SINGLY REINFORCED BEAM (IS 456:2000) ━━

Inputs: Width b (mm), Effective depth d (mm), fck, fy,
Factored moment Mu (kN·m), Factored shear Vu (kN)

Logic:
Step 1: Limiting moment of resistance
Mu_lim = 0.138 × fck × b × d² (Fe415)
= 0.133 × fck × b × d² (Fe500)
Step 2: Check if Mu ≤ Mu_lim (else needs doubly reinforced)
Step 3: Solve quadratic for Ast:
Mu = 0.87×fy×Ast×d × (1 - Ast×fy/(b×d×fck))
Step 4: Min steel = 0.85×b×d/fy (Cl. 26.5.1.1)
Step 5: Max steel = 0.04×b×D
Step 6: Choose bar configuration (e.g., 3-16φ = 603 mm²)
Step 7: Shear design — compute τv = Vu/(b×d)
Find τc from IS 456 Table 19 based on pt and fck
If τv > τc: design stirrups
Vus = Vu - τc×b×d
Spacing = (0.87×fy×Asv×d) / Vus
Min stirrup spacing check per Cl. 26.5.1.6
Step 8: Side face reinforcement if D > 750mm (Cl. 26.5.1.3)
Step 9: Check deflection (Cl. 23.2.1) with modification factors

━━ MODULE 1.3 — DOUBLY REINFORCED BEAM ━━

Inputs: b, D, d, d' (cover to compression steel), fck, fy, Mu

Logic:
Step 1: Compute Mu_lim
Step 2: If Mu > Mu_lim: Mu2 = Mu - Mu_lim
Step 3: fsc = stress in compression steel (700×(xu_max-d')/xu_max,
limited to 0.87fy for Fe415, per IS 456 stress-strain)
Step 4: Asc = Mu2 / (fsc × (d - d'))
Step 5: Ast2 = Asc × fsc / (0.87 × fy)
Step 6: Ast_total = Ast1 + Ast2 (where Ast1 = balanced section steel)
Step 7: All IS checks as in singly reinforced beam

━━ MODULE 1.4 — T-BEAM DESIGN ━━

Inputs: bw (web width), Df (flange thickness), span L, spacing of beams,
d, fck, fy, Mu

Logic:
Step 1: Effective flange width bf per IS 456 Cl. 23.1.2:
bf = Lo/6 + bw + 6Df (T-beam)
bf = Lo/12 + bw + 3Df (L-beam)
bf ≤ actual available width
Step 2: Check if NA lies in flange (Mu ≤ Mu_flange):
Mu_flange = 0.87×fy×Ast×(d - 0.42×Df)
if Df/d ≤ 0.2: treat as rectangular beam of width bf
Step 3: If NA in web: compute using T-section moment formula
Step 4: Solve for Ast

━━ MODULE 1.5 — SHORT COLUMN DESIGN ━━

Inputs: b, D (cross-section), effective length lex, ley,
Pu (factored axial load, kN), Mux, Muy (factored moments, kN·m),
fck, fy, cover

Logic:
Step 1: Slenderness check: lex/D ≤ 12 (short column condition)
If > 12: flag as slender, still compute with note
Step 2: For axial only: Pu_capacity = 0.4×fck×Ac + 0.67×fy×Asc
Rearrange for Asc required
Step 3: Min steel = 0.8% Ag, Max = 6% Ag (Cl. 26.5.3.1)
Step 4: For combined bending: use interaction diagram approach
Compute P0, Mb0, compute ratios Pu/Puz, Mux/Mux1, Muy/Muy1
Check (Mux/Mux1)^αn + (Muy/Muy1)^αn ≤ 1.0 (IS 456 Cl. 39.6)
where αn = 1.0 to 2.0 based on Pu/Puz
Step 5: Lateral ties: dia ≥ φmain/4 or 6mm whichever greater
Spacing ≤ least of: least lateral dim, 16×φmain, 300mm

━━ MODULE 1.6 — ISOLATED FOOTING DESIGN ━━

Inputs: Column size (bx × by), Column load P (kN), Column moment M (kN·m),
Safe Bearing Capacity q (kN/m²), fck, fy, unit weight of soil γ (kN/m³),
depth of foundation Df (m)

Logic:
Step 1: Net SBC = q - γ×Df - 0.5×25 (self weight of footing assumed)
Step 2: Area required = P / net SBC, size footing (square or rectangular)
Step 3: Check eccentricity: e = M/P, e ≤ L/6
Step 4: Net upward pressure qu = Pu / (L×B) (factored)
Step 5: Critical section for BM at face of column:
Mu = qu × B × ((L-bx)/2)² / 2
Step 6: Depth from BM, also check for one-way shear and
two-way (punching) shear — both critical
Step 7: Two-way shear: critical perimeter at d/2 from column face
τv = Vu / (bo × d), compare with ks×τc (IS 456 Cl. 31.6.3.1)
Step 8: Ast design, check transfer of load at column base (Cl. 34.4)
Step 9: Dowel bars / development length check

━━ MODULE 1.7 — DOG-LEGGED STAIRCASE ━━

Inputs: Floor height H (m), Going G (mm), Riser R (mm),
Width of stair (m), fck, fy, LL on stair (kN/m²)

Logic:
Step 1: Number of risers = H/R, check 150 ≤ R ≤ 200mm
Step 2: Number of goings = risers - 1, G ≥ 250mm
Step 3: Check: 2R + G = 550 to 700mm
Step 4: Span of waist slab (inclined), horizontal span
Step 5: Loads on inclined slab:
Self weight = 25 × t × (√(R²+G²)/G)
Finish = 1.0 kN/m² on plan area
LL per IS 875 Part 2
Step 6: Total factored wu, compute Mu
Step 7: Design waist slab (same as one-way slab)
Step 8: Landing slab design (treated separately)

━━ MODULE 1.8 — CANTILEVER RETAINING WALL (IS 456:2000 / IS 3370) ━━

Inputs: Height of retained earth H (m), SBC q (kN/m²),
Unit weight of soil γ (kN/m³), Angle of internal friction φ (°),
Surcharge q_s (kN/m²), fck, fy

Logic:
Step 1: Rankine's Ka = (1-sinφ)/(1+sinφ)
Step 2: Pressure distribution on stem
Step 3: Design stem (as vertical cantilever): M at base of stem
Step 4: Proportion base slab: L ≈ 0.4 to 0.6H typically
Step 5: Check overturning: FOS ≥ 1.5 (taking moments about toe)
Step 6: Check sliding: FOS ≥ 1.5, μ=0.5 typically
Step 7: Check bearing pressure ≤ SBC at toe and heel
Step 8: Design heel slab, toe slab as cantilever beams
Step 9: Shear key design if needed

━━ MODULE 2.1 — SHEAR & STIRRUP DESIGN ━━

Inputs: b, d, Vu, fck, fy_stirrup, Ast provided, bar configuration

Logic per IS 456 Cl. 40:
Step 1: τv = Vu / (b×d)
Step 2: Find pt = 100×Ast/(b×d)
Step 3: τc from IS 456 Table 19 (implement full table as lookup with interpolation)
Step 4: τc_max from IS 456 Table 20
Step 5: If τv ≤ τc: only minimum stirrups per Cl. 26.5.1.6
Step 6: If τc < τv ≤ τc_max: Vus = (τv - τc)×b×d
For 2-legged 8φ stirrups: Sv = 0.87×fy×Asv×d / Vus
Round down to nearest 25mm
Step 7: Min Sv = 0.75d or 300mm
Step 8: If τv > τc_max: redesign section (show error)

━━ MODULE 2.2 — DEVELOPMENT LENGTH ━━

Inputs: Bar dia φ, fck, fy, type (tension/compression),
Surface condition (plain/deformed)

Formula:
Ld = φ × σs / (4 × τbd)
where τbd from IS 456 Table 5 (implement full table)
For tension deformed bars: τbd × 1.6 (Cl. 26.2.1.1)
For compression: Ld (compression) = 0.8 × Ld (tension)

Also compute for standard hook reduction (Cl. 26.2.2.1)

━━ MODULE 2.3 — DEFLECTION CHECK ━━

Inputs: Span L, d provided, fs (steel stress at service),
pt_required, pt_provided, modification factors

Logic per IS 456 Cl. 23.2.1:
Basic L/d ratio: 7 (cantilever), 20 (simply supported), 26 (continuous)
Modification factor kt for tension reinforcement (IS 456 Fig. 4)
Modification factor kc for compression reinforcement (IS 456 Fig. 5)
Modification factor kf for flanged beams (IS 456 Fig. 6)
Allowable L/d = basic × kt × kc × kf
Check actual L/d ≤ allowable

━━ MODULE 3.1–3.5 — STEEL DESIGN (IS 800:2007 — Limit State) ━━

3.1 Tension Member:
- Tdg = Ag × fy / γm0 (gross section yielding, Cl. 6.2)

3.3 Beam Design:

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every calculator must have an Export button in the results panel.
Implement TWO export options:

OPTION A — Export to PDF
Use the browser's window.print() API with a dedicated print stylesheet:
- @media print: hide sidebar, hide input panel, show only results
- Print header: App logo + Calculator name + Date + Time
- Print footer: "Generated by Stellar Civil | For reference only.
Verify with qualified engineer."
- All tables print with borders
- Step-by-step table must break cleanly across pages
- Force page break before compliance checklist

OPTION B — Export to CSV (for BBS and summary tables)
- Generate CSV string from result tables
- Trigger download using Blob + URL.createObjectURL
- Filename: "StellarCivil_[ModuleName]_[Date].csv"

Both export buttons must be disabled until calculations are run.
Show "⬇ Export PDF" and "⬇ Export CSV" buttons side by side.

SECTION  — UX & INTERACTION REQUIREMENTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. INPUT VALIDATION (real-time, on every input change):
- Show red border + error message below if value out of range
- Show green checkmark when valid
- Disable "Calculate" button if any input is invalid
- Show count of invalid fields in button: "Fix 2 errors"

2. CALCULATE BUTTON BEHAVIOR:
- Show loading spinner for 300ms (simulated) before showing results
- Scroll smoothly to results after calculation
- Button text changes to "Recalculate" after first run

3. CLEAR/RESET:
- "Clear All" restores all inputs to default values
- Clears results panel
- Asks for confirmation via a small inline confirmation (not JS alert)

4. STICKY SUMMARY:
- After calculation, a small sticky bar at the bottom of viewport shows
the 3 most critical results (e.g., D=175mm, Ast=628mm²/m, Status: ✅ OK)
- This persists as user scrolls through steps

5. INPUT MEMORY (localStorage):
- Save last used inputs for each module in browser localStorage
- On revisiting a module, pre-fill with last used values
- Show "Restored last session" toast notification

6. KEYBOARD SHORTCUTS:
- Enter key in any input → triggers calculation
- Ctrl+P → triggers PDF export
- Esc → collapses results panel
- Ctrl+R → reset inputs

7. TOOLTIPS:
- Every input field has a ? tooltip icon
- Hover shows: parameter name, IS Code clause reference, typical value range
- Tooltips implemented in pure CSS (no JS required)

8. RESPONSIVE BEHAVIOR:
- Desktop (>1200px): sidebar + two-column input/output
- Tablet (768–1200px): sidebar collapses, single column
- Mobile (<768px): hamburger menu, full-screen calculator, bottom export bar

9. ERROR HANDLING:
- If calculation produces NaN or Infinity: show "Calculation error —
check inputs" in a red card. Do not show partial results.
- If section fails (e.g., depth insufficient): show clearly with redesign suggestion

10. DARK MODE:
- Toggle in top-right corner (moon/sun icon)
- Preference saved to localStorage
- Smooth CSS transition (transition: background 0.3s, color 0.3s)

SECTION 9 — REFERENCE MODULES (Non-Calculator)
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

MODULE 6.2 — Material Properties Table
A searchable, sortable HTML table showing concrete and steel properties.
Columns for concrete: Grade, fck, fck_cube, Ec, εcu, τbd, etc.
Columns for steel: Grade, fy, fu, Es, elongation

MODULE 6.3 — Rebar Reference
Table: dia, area, perimeter, weight/m, areas for 1–10 bars,
common spacings (75/100/125/150/175/200/225/250mm) → area per meter

MODULE 6.4 — IS Code Cheatsheet
Quick-reference card for the most commonly used clauses:
- Cover requirements (IS 456 Table 16)
- Min/max steel ratios
- Partial safety factors (γf for loads, γm for materials)
- Load combinations (IS 456 / IS 1893)
- Standard hooks and bends
- Development length quick table by dia and grade

MODULE 6.1 — Unit Converter
Bidirectional converters for:
- Force: N ↔ kN ↔ tf ↔ kip ↔ lbf
- Stress: N/mm² ↔ kN/m² ↔ MPa ↔ ksi ↔ psi
- Length: mm ↔ cm ↔ m ↔ ft ↔ inch
- Moment: kN·m ↔ tf·m ↔ kip·ft ↔ lbf·in
- Area: mm² ↔ cm² ↔ m² ↔ in²
- Moment of Inertia: mm⁴ ↔ cm⁴ ↔ in⁴

. PRECISION:
- Intermediate: work in base units (N, mm) throughout, convert only for display
- Display: round to engineering convention (D to nearest 5mm, Ast to 1 mm²,
spacing to nearest 25mm, loads to 2 decimal places)

5. VALIDATION LIBRARY:
Build a centralized validate(value, min, max, name) function that:
- Returns {valid: bool, message: string}
- Used consistently for all inputs across all modules
